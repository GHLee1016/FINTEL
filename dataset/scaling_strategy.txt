================================================================================
  FINTEL — Feature별 Scaling 전략 (ML / DL 전용)
  적용 모델: 선형 (Ridge / ElasticNet / Huber)
  비적용 모델: 트리 (LightGBM / XGBoost) — raw 값 입력
              금융모형 (GARCH / HAR-RV) — 자체 단위 사용
================================================================================

  · 4개 그룹으로 분류: log_robust / robust / standard / no_scaling
  · 모든 feature를 명시적으로 분류 (default fallback에 의존하지 않음)
  · 타깃 RV_target은 명시적 제외 (예측값을 RV 단위 그대로 출력)
  · Walk-forward expanding 시: 매 refit마다 scaler를 train 데이터로 새로 fit
    (test 데이터의 통계가 train scaling에 누설되지 않도록)


================================================================================
  [그룹 1] log_robust  (5개)
================================================================================

  변환: log1p(clip(x, 1e-8, None))  →  RobustScaler
  대상: 양수 + 꼬리 두꺼운 분포의 변동성 변수

    RV_d              일별 실현변동성
    RV_w              주간 실현변동성 (HAR-w)
    RV_m              월간 실현변동성 (HAR-m)
    semivariance      준분산 (downside risk)
    parkinson_rv      Parkinson high-low 기반 RV 추정치


================================================================================
  [그룹 2] robust  (6개)
================================================================================

  변환: RobustScaler 단독 (이상치에 강건한 중앙값/IQR 기반)
  대상: 변동률·범위 — 음수 가능, 이상치 많음

    hl_range          고저 변동폭 (High-Low Range)
    log_return        로그 수익률
    gold_chg          금 가격 변동률
    wti_chg           WTI 원유 변동률
    corn_chg          옥수수 변동률
    epu               경제정책 불확실성 지수 (EPU)


================================================================================
  [그룹 3] standard  (15개)
================================================================================

  변환: StandardScaler (z-score: (x − mean) / std)
  대상: 정규분포 가정이 합리적인 거시·모멘텀·spillover

    fx_level          환율 수준
    fx_change         환율 변동률
    rate_3y           3년 국채 금리
    rate_10y          10년 국채 금리
    rate_spread       장단기 금리 스프레드
    policy_rate       정책 금리
    crb_chg           CRB 원자재 지수 변동률
    natgas_chg        천연가스 변동률
    momentum_1w       1주 모멘텀
    momentum_1m       1개월 모멘텀
    momentum_3m       3개월 모멘텀
    momentum_6m       6개월 모멘텀
    spillover_SP500   S&P 500 전일 수익률 spillover
    spillover_Nikkei  Nikkei 225 전일 수익률 spillover
    spillover_KOSPI   KOSPI 전일 수익률 spillover


================================================================================
  [그룹 4] no_scaling  (3개 — 모두 명시적)
================================================================================

  변환: 없음 (변환 자체를 스킵)
  대상: 이미 정규화된 값 또는 부호 의미가 보존되어야 하는 변수

    neg_return        음수 수익률 (downside, 부호 의미 보존)
    weekday_sin       요일 sin 인코딩 (이미 [-1, 1] 범위)
    weekday_cos       요일 cos 인코딩 (이미 [-1, 1] 범위)


================================================================================
  특수 처리: 타깃 변수 (명시적 제외)
================================================================================

    RV_target         예측 타깃 — scaling 적용 안 함

  · feature_cols에 RV_target이 들어가지 않도록 데이터 준비 단계에서 가드 필요.
  · 모델 출력은 RV_target과 동일 단위 (% 분산)로 직접 비교됨.


================================================================================
  요약 표
================================================================================

  그룹             개수    변환
  --------------   -----   --------------------------------------------
  log_robust          5    log1p(clip) + RobustScaler
  robust              6    RobustScaler
  standard           15    StandardScaler
  no_scaling          3    (변환 없음)
  타깃 (제외)         1    (변환 없음, feature_cols에서 명시적 배제)
  --------------   -----   --------------------------------------------
  합계               30    (29 feature 후보 + 1 target)

  ※ 시장당 실제 사용되는 feature 수는 EXTENDED tier 기준 28개
    (29개 후보 중 자기 시장의 spillover 1개는 CSV에 없으므로 자동 제외).


================================================================================
  적용 정책
================================================================================

  · 선형 모델: 위 정책 그대로 적용. CustomScaler를 train으로 fit → test 에 transform.
  · 트리 모델: scaling 자체 생략. 트리는 monotonic 변환에 불변이라 효과 없음.
  · 금융모형: 적용 안 함 (자체 단위로 동작).
  · 결측치 처리: 본 정책은 scaling 단계만 정의. 결측 처리(forward fill, dropna 등)는
    데이터 로딩·전처리 단계에서 별도로 수행.


================================================================================
  설계 원칙 — 안전장치 3가지
================================================================================

  1. Spillover 3종 (SP500 / Nikkei / KOSPI)은 모두 standard 그룹.
     → 의미와 단위(전일 수익률)가 같은 변수는 같은 scaling을 적용해야 일관됨.
        만약 일부만 robust로 두면 동일 정보가 모델별로 다른 분포로 들어가 비교가 흐려짐.

  2. weekday_sin / weekday_cos를 명시적으로 no_scaling 그룹에 등록.
     → default fallback("정의 안 된 컬럼 = NoScaling")에 의존하지 않음. 향후
        SCALING_CONFIG를 손볼 때 이 두 변수가 누락되어도 즉시 에러 (strict 모드)로 잡힘.

  3. RV_target을 어떤 그룹에도 두지 않고, scaler 호출 시 명시적 가드.
     → 타깃이 실수로 feature_cols에 들어가 학습되는 사고를 방지.
        CustomScaler.fit_transform / transform이 RV_target 발견 시 ValueError로 즉시 차단.
